# kafka-loader

`kafka-loader` — небольшая инфраструктурная утилита для генерации контролируемого producer-side трафика в Kafka из JSONL-файлов.

Приложение читает properties-конфиг, загружает записи из локального `.jsonl` файла и непрерывно отправляет их в один или несколько Kafka-топиков. Интенсивность задаётся через `eps.per.topic` и `threads.per.topic`, поэтому утилита подходит для smoke-проверок, проверки подключения, валидации SSL и воспроизводимой генерации producer-side нагрузки на Linux-хостах.

Сейчас поддерживаются следующие варианты подключения:

- `PLAINTEXT`
- `SSL` только с проверкой сертификата сервера
- `SSL` с mutual TLS (mTLS)

Задача утилиты — дать простой и предсказуемый Kafka producer, который настраивается через файл и может запускаться на Linux-хосте как отдельный бинарник без необходимости ставить полноценное Python-окружение на целевую машину.

Основные параметры конфигурации:

- адреса Kafka bootstrap servers и список топиков
- путь к исходному JSONL-файлу
- скорость отправки сообщений на топик
- количество рабочих потоков на топик
- producer-настройки, такие как `acks`, `linger.ms`, `batch.size`, `buffer.memory` и тип компрессии
- пути к truststore и keystore для защищённого подключения к Kafka

Во время работы loader читает непустые строки из входного файла в память и затем переиспользует их по кругу при отправке в Kafka. За счёт этого удобно воспроизводить поток сообщений на фиксированном наборе данных во время инфраструктурных проверок, валидации окружения и тестовых прогонов.
